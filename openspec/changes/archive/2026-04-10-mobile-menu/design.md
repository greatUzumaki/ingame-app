## Context

Приложение использует Next.js App Router. Navbar — `"use client"`, sticky top-0, z-50. Таббар должен быть поверх контента (z-50), фиксированный снизу, видимый только на мобильных (`md:hidden`). Профиль ссылается на `/auth/login` если пользователь не вошёл, или на будущую страницу профиля.

## Goals / Non-Goals

**Goals:**
- Нижний таббар с 5 табами, всегда виден на мобильных
- Активный таб: яркий цвет иконки + лейбла + анимированная точка/полоска сверху
- Нажатие: `scale(0.85)` bounce через CSS transition
- Фон таббара: `bg-[var(--color-brand-dark)]/95 backdrop-blur-md`, border-top

**Non-Goals:**
- Реальные страницы "Новости" и "Магазин" — только заглушки
- Анимация перелёта индикатора между табами (сложно без JS измерений — просто fade/opacity)

## Decisions

### Компонент — `BottomNav.tsx`, `"use client"`
Читает `usePathname()` для активного состояния. Иконки — inline SVG. `useFavourites` не нужен.

### Анимация нажатия — CSS `active:scale-90`
Tailwind `active:scale-90 transition-transform duration-150` на каждой кнопке. Не требует JS.

### Активный индикатор — точка над иконкой
`w-1 h-1 rounded-full bg-[var(--color-brand-accent)]` с `opacity-0 scale-0 → opacity-100 scale-100 transition-all` когда таб активен.

### Профиль — условный href
Если `user` из `useAuth()` — ссылка на `/profile` (заглушка), иначе `/auth/login`.

### Отступ снизу для контента
`<main>` в `layout.tsx` добавить `pb-16 md:pb-0` — 64px = высота таббара.

### Бургер в Navbar
Добавить `hidden` к бургер-кнопке: `md:hidden` → оставить только для `sm`... нет, убрать совсем на мобильных — таббар заменяет. Бургер остаётся скрытым (уже `md:hidden` для десктопа, добавить `hidden` — убрать бургер полностью, так как таббар берёт на себя мобильную навигацию).

Альтернатива: оставить бургер как есть для авторизации. Проще — убрать бургер, добавить auth в таббар через иконку Профиля.

## Risks / Trade-offs

- **Заглушки /news и /shop**: Пользователь увидит пустые страницы. Добавить "скоро" сообщение.
- **Safe area (notch)**: На iPhone с notch нижняя полоска перекроет таббар. Добавить `pb-safe` или `env(safe-area-inset-bottom)` через inline style.
- **Бургер убран**: Авторизация на мобильных теперь только через таб "Профиль".
